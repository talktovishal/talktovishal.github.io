Learning by doing

Attention by hand: make "it" look back at "cat".

This draft turns the attention layer into a workbook. You compute the numbers, check each one, open hints when needed, and reveal the solution only when you want the training wheels.

Our entire chapter will keep one sentence on the table: Sarah fed the cat because it. The question is wonderfully concrete: when the model reaches it, how does the attention layer pull useful information from cat into that stream?

We will not jump straight to a giant matrix. First you compute one score. Then you turn scores into weights. Then you blend values. Only after that do we zoom out to the full attention matrix, the causal mask, the KV cache, grouped-query attention, and FlashAttention. Same numbers, wider view.

Lab 1 One attention head, one query row, every number visible.
Lab 2 The whole score matrix, then the causal mask.
Lab 3 KV cache, grouped-query attention, and FlashAttention bookkeeping.

Before you start, a few prerequisites are in order. You only need the pieces below, and each one gets a local refresher on this page.

  • Vectors. A token is represented as a short row of numbers.
  • Dot products. Multiply matching channels, then add.
  • Softmax. Turn scores into positive weights that add to 1.
  • Matrix shape. Rows and columns tell you what can multiply what.

What you do not need yet: training. These Q/K/V numbers are hand-picked so the mechanism is visible.

1. Math kit

The rules we will use, with the "why" tucked inside.

This is intentionally compact. The card gives the rule; the collapsed proof shows why it works. You can skip the proof on a first pass and open it exactly when a step feels like it came from nowhere.

Dot product

\(q \cdot k\) means multiply matching channels and add the products.

Show why it measures match

If two vectors put large numbers in the same channels, their products are large and positive.

[a,b,c,d] \cdot [w,x,y,z] = aw + bx + cy + dz

In attention, the query is the question and the key is the label. A large dot product means the label matches the question.

Scaling by sqrt(d)

Attention uses \(q \cdot k / \sqrt{d_k}\) so scores stay in a softmax-friendly range.

Show why we scale

A dot product adds \(d_k\) products. Wider vectors tend to make larger raw sums even when the match quality has not really changed.

raw score grows with vector width scaled score = raw score / sqrt(d_k)

The scaling keeps softmax from becoming too spiky just because the vectors are wider.

Softmax

Softmax turns scaled scores into weights: exponentiate, add, then divide.

Show the steps
weight_i = exp(score_i) / sum_j exp(score_j)

Exponentials make every weight positive. Dividing by the sum makes all weights add to 1.

sum_i weight_i = sum_i exp(score_i) / sum_j exp(score_j) = 1

Causal mask

Future scores become \(-\infty\) before softmax, so their weights become 0.

Show why future weight is zero
exp(-infinity) = 0 future weight = 0 / denominator = 0

The future token stays in the grid, but it contributes nothing.

Weighted sum

The attention output is each value vector multiplied by its attention weight, then added.

Show why values blend
output = alpha_1 v_1 + alpha_2 v_2 + ...

If a token gets a small weight, its value barely contributes. If it gets a large weight, its value dominates the output.

Matrix shapes

For \(N\) tokens, \(QK^\top\) is an \(N \times N\) score grid.

Show the shape logic
Q shape: N x d_k K^T shape: d_k x N QK^T shape: N x N

Rows are query tokens. Columns are key tokens.

KV cache

During generation, old keys and values can be reused because past tokens do not change.

Show what is saved
without cache: remake K and V for the whole prefix with cache: make K and V only for the new token

The new query still scores against every cached key, but the old key/value projections are not recomputed.

FlashAttention

FlashAttention computes the same softmax attention while streaming through tiles.

Show what stays the same
normal: materialize scores -> softmax -> multiply V flash: tile scores -> keep running max/sum/output

The result is exact attention. The win is less slow memory traffic, not a different formula.
